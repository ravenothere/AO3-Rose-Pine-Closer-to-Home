About

This is a modified version of Rosé Pine Site Skin by BlackBatCat — all credit for the original goes to them.

This version restores some of the default AO3 interface aesthetics, bringing back a little of what makes AO3 feel like AO3 while keeping everything that makes the theme beautiful.🌷

The changes are mostly subtle — icon shapes and colors closer to AO3's defaults, softer pastel tag colors, title decorations removed, and a few layout tweaks.
If you're curious about the full list of differences, the details are on the GitHub page.

Installation

To use this skin on AO3, you'll be creating a small set of linked skins — think of it like building blocks that stack on top of each other. The base sets up the layout, the theme adds the colors, and the tablet and mobile skins handle smaller screens. Then you chain them all together at the end.

Important: You must choose between the Normal Layout or the Card Layout before you begin. Once you pick one, use all the corresponding files for that layout throughout the steps. Do not mix files from different layouts.

0. Choose Your Layout

Before you begin, decide which layout you prefer:

  • Normal Layout: Uses CTH_base.css, CTH_tablet.css, and CTH_mobile.css
  • Card Layout: Uses CTH_base_alt.css, CTH_tablet_alt.css, and CTH_mobile_alt.css

Check the previews above to see which style you prefer. Both layouts use the same theme files, so you can pick any theme color regardless of which layout you choose.

1. Create the Base Skin

  • Go to Dashboard → Skins → Create Site Skin.
  • Give it a name like [XYZ] Rosé Pine CTH - Base — replace XYZ with your username or initials.
  • Paste the contents of your chosen base file into the CSS field:
    • Normal Layout: CTH_base.css
    • Card Layout: CTH_base_alt.css
  • Under Advanced, set it to Parent Only.
  • Click Submit.

2. Create a Theme Skin

This is where you pick your colors!

  • Create a skin named something like [XYZ] Rosé Pine Dawn CTH or [XYZ] Rosé Pine Moon CTH.
  • Paste the contents of your chosen theme file:
    • Rosé Pine Dawn — light theme
    • Rosé Pine Moon — dark theme
    • Rosé Pine — dark theme
  • Set it to Parent Only and click Submit.

2a. (Optional) Auto Light/Dark Mode Switching

Want the skin to automatically switch between light and dark based on your device's settings?

  • Create a second theme skin — for example, Dawn for light and Moon for dark.
  • Paste the CSS code from your second theme, set it to Parent Only.
  • On your dark theme skin, go to Advanced → Choose @media and select (prefers-color-scheme: dark).
  • On your light theme skin, select (prefers-color-scheme: light).
  • When you get to Step 5, add both theme skins to the parent chain — they'll switch automatically.

Known AO3 bug: The skin editor sometimes doesn't save @media settings properly when editing an existing skin. If you run into this, delete the skin and create a new one with the correct settings.

3. Create the Tablet Skin

Important: The tablet and mobile skins are required — the theme won't display correctly without them.

  • Create a skin named [XYZ] Rosé Pine CTH - Tablet.
  • Paste the correct tablet file for your layout:
    • Normal Layout: CTH_tablet.css
    • Card Layout: CTH_tablet_alt.css
  • Under Advanced → Choose @media, select only screen (max-width: 62em).
  • Set to Parent Only and submit.

4. Create the Mobile Skin

  • Create a skin named [XYZ] Rosé Pine CTH - Mobile.
  • Paste the correct mobile file for your layout:
    • Normal Layout: CTH_mobile.css
    • Card Layout: CTH_mobile_alt.css
  • Under Advanced → Choose @media, select only screen (max-width: 42em).
  • Set to Parent Only and submit.

5. Chain Everything Together

  • Create one final skin named something like [XYZ] Rosé Pine CTH - Default.
  • In the CSS field, paste .rose-pine { opacity: 1; } — this is just a placeholder so AO3 lets you save.
  • Under Advanced → Parent Skins, add the skins you created in this order:
    1. [XYZ] Rosé Pine CTH - Base
    2. Single theme: your theme skin (e.g. [XYZ] Rosé Pine Dawn CTH)

      OR for auto switching: add both of your theme skins — order doesn't matter since they use @media queries.
    3. [XYZ] Rosé Pine CTH - Tablet
    4. [XYZ] Rosé Pine CTH - Mobile
  • Click Submit, then Use.

Customization

Tip: AO3's code editor doesn't save annotations (the labels for code sections), so if you want to edit a skin later, copy it to a text editor first to make your changes, then paste it back.

Fonts

To change the font, open your theme skin and press Ctrl+F to search for font-family. You'll find four blocks — one for the main site font, one for headings, one for code, and one for works. Replace the first font name in each with your preferred font, keeping the fallbacks:

font-family: YourFont, Apfel Grotezk, Figtree, Bitter, Helvetica, Arial, sans-serif;

Important: The font must be installed on your device to display. If it isn't, the browser will fall back to the next one in the list.

On mobile: AO3 can only display fonts installed on your device. The recommended approach on Android is to use the Stylus extension on Firefox — it can load fonts directly without needing them installed system-wide. The AO3: Rosé Pine Fonts userstyle has the fonts for this skin ready to go. If you want to use custom fonts on mobile, see my Custom Fonts for Mobile guide.

Text Size

In your theme skin, find these two variables near the top:

--txt-size-main: 110%;   /* main text size */
--txt-size-work: 115%;   /* work text size */

--txt-size-main controls text across the whole site (navigation, blurbs, forms, etc.), while --txt-size-work is just for work content. Increase the percentage for larger text or decrease for smaller. Going above 130% for --txt-size-main may cause minor layout issues.

Tag Colors

Want to change up the tag colors? The variables are in the TAG COLORS section near the top of your theme file:

/*>========== TAG COLORS ==========<*/

  /* BACKGROUND */
  --tag-default-bg:      #f0f8f9;
  --tag-fandom-bg:       #d8edc2;
  --tag-warning-bg:      #fadaae;
  --tag-ship-bg:         #f1bab8;
  --tag-character-bg:    #b8d9e6;
  --tag-freeform-bg:     var(--tag-default-bg);
  --tag-splash-bg:       #F6ECE3;

  /* TEXT */
  --tag-default-txt:     #464261;
  --tag-fandom-txt:      #464261;
  --tag-warning-txt:     #464261;
  --tag-ship-txt:        #464261;
  --tag-character-txt:   #464261;
  --tag-freeform-txt:    #464261;

Just swap out the hex color values with whatever you like! For example, if you want purple fandom tags, change --tag-fandom-bg: #d8edc2; to --tag-fandom-bg: #c4a7e7;. You can also tweak the text color for each tag type using the --tag-*-txt variables.

Note: Leave --tag-splash-bg alone — it's specifically for the splash page and doesn't affect regular tag colors.

Platonic Ship Tags

Want platonic ships (relationships with the & symbol) to stand out with their own color? There are pre-made add-on skins with this already built in:

  • CTH_Platonic-tags-light.css — for Dawn / light theme
  • CTH_Platonic-tags-dark.css — for Moon, Rosé Pine / dark themes

Add whichever matches your theme to the parent chain after your theme skin. If you're using auto light/dark switching, add both.

Recommended to use this alongside AO3: Reorder Ship Tags addon.
